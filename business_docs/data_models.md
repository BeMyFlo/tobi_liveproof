# Tobi LiveProof Data Models (MongoDB)

LiveProof uses Mongoose to manage its MongoDB schemas. Here's a summary of the core data entities.

## 1. User (`User.ts`)
The central entity representing a store owner who uses LiveProof.
-   `email`, `password`: Standard authentication.
-   `apiKey`: A unique string (prefixed with `tlp_`) used by `live.js` to identify the site owner.
-   `plan`: `free`, `pro`, or `premium`. Controls access to features.
-   `widgets`: A collection of settings for each widget type (Viewers, Purchases, Exit Intent, etc.).
    -   Each widget has `enabled`, `position`, `style`, `template`, `delay`, and specific settings like `discountCode`.

## 2. Campaign (`Campaign.ts`)
Used for Flash Sale events.
-   `userId`: References the site owner.
-   `name`, `type`, `status`: Management of the sale event.
-   `startTime`, `endTime`: Visibility schedule.
-   `products`: Array of product objects (name, image, url, originalPrice, salePrice).
-   `isGlobal` / `targetUrls`: Controls where the flash sale bubble appears.

## 3. Event (`Event.ts`)
Logs interactions with widgets for analytics.
-   `apiKey`: Identifies the project.
-   `widgetType`: E.g., `viewers`, `purchases`.
-   `eventType`: `view` or `click`.
-   `metadata`: Contextual information (URL, etc.).

## 4. Visitor (`Visitor.ts`)
Tracks individual shoppers across visits for personalization.
-   `visitorId`: Unique UUID generated by `live.js` and stored in the browser's `localStorage`.
-   `visitCount`, `firstVisit`, `lastVisit`: Engagement metrics.
-   `recentPages`: List of URLs visited.
-   `metadata`: IP, User Agent, and geo-location (Country, City) if available.

## 5. Purchase (`Purchase.ts`)
Records of actual customer purchases to be shown in "Social Proof" popups.
-   `apiKey`: Identifies the project.
-   `productName`: Name of the purchased item.
-   `customerName`: "Someone" or the actual name (if provided via API).
-   `customerLocation`: Location of the purchaser for social proof accuracy.

## 6. Payment (`Payment.ts`)
Tracks subscription and billing history via SePay.
-   `orderCode`, `memo`: Used for identifying bank transfers.
-   `status`: `PENDING`, `PAID`, or `CANCELLED`.
-   `paymentLinkId`: The transaction reference.
